Update the existing Awaaz Setu prototype with two important safety and trust features:

1. Controlled Withdraw / Revoke Report feature for L1 and L2 cases.
2. Optional DigiLocker-style Trust Verification layer to reduce fake reporting without breaking user anonymity.

Important:
This is a demo prototype only.
Do not add real DigiLocker API.
Do not add real Aadhaar verification.
Do not store Aadhaar number.
Do not add backend.
Use mock UI, mock verification status, localStorage, and demo-only logic.

Project Positioning Reminder:
Awaaz Setu is not a police portal, not a court, not an FIR system, not an emergency service, and not a public complaint wall.

PART 1: ADD WITHDRAW / REVOKE REPORT FEATURE
============================================

Purpose:
Some users may submit a report in anger, fear, confusion, misunderstanding, or emotional pressure. Later they may realize they do not want further action. The platform should allow controlled withdrawal for low-risk cases, but serious safety cases must not be directly deleted or withdrawn without review.

Use wording:
Use “Withdraw Report” instead of “Delete Report”.
Use “Withdrawn by Reporter” as status.
Do not permanently delete reports in demo mode.
Only update status, timeline, and audit log.

---

## A. Track Report Page Update

On the Token-Based Tracking page, after user enters a valid token and report details appear, add these user action buttons:

1. Withdraw Report
2. Correct My Report
3. Pause Support Request

Button visibility rules:

Show “Withdraw Report” only when:

* Severity is L1 or L2
* Report is not already routed to counselor/NGO/district committee
* Report is not resolved/closed
* Report does not contain self-harm, suicide, child abuse, sexual assault, physical danger, or emergency keywords

For L3 or L4:
Do not allow direct withdrawal.
Instead show:
“Request Withdrawal Review”

Show message:
“This report may involve higher safety risk. A withdrawal request will be reviewed for safety before any action is stopped.”

PART 2: ADD OPTIONAL DIGILOCKER-STYLE TRUST VERIFICATION
========================================================

Purpose:
Fake reporting is a concern. But if DigiLocker/Aadhaar verification is made mandatory, real victims may feel scared and leave the platform. Awaaz Setu should follow:

“Anonymous First, Verified Later”

Meaning:
User can submit report anonymously first.
Verification should be optional.
Verification should be used only as a trust layer, not as public identity display.

Important:
Do not integrate real DigiLocker API.
Create a mock DigiLocker verification UI only.
Do not ask for Aadhaar number.
Do not show Aadhaar field.
Do not store Aadhaar data.
Do not use words like “KYC mandatory” or “Aadhaar required”.

---

## A. Add Verification Choice in Report Form

In the Anonymous Report Form, add a step:

“Optional Trust Verification”

Show two cards:

Card 1:
Continue Anonymously
Description:
“Submit safely without name, phone, email, Aadhaar, or login.”

Card 2:
Add Trust Verification
Description:
“Optional verification can improve report credibility during support routing. Your identity will remain hidden from public view and normal dashboard view.”

Recommended badge:
“Optional”

Default selected option:
Continue Anonymously

---

## B. Safe Wording Near Verification

Add this text before verification button:

“DigiLocker-style verification is optional in this demo. Awaaz Setu will not show your Aadhaar number in the report. Your identity will not be visible to the public, respondent, or normal dashboard view. Verification only helps reduce misuse and improve trust during support routing. You can continue anonymously.”

Button text:
“Verify Safely — Demo”

Do not use:

* Aadhaar Verify
* KYC Required
* Identity Mandatory
* Aadhaar Authentication Required

---

## G. Fake Report Risk Controls Page / Section

Add a section called:
“How Awaaz Setu Reduces Misuse”

Cards:

1. Optional Trust Verification
   Reduces fake reporting without forcing Aadhaar on first step.

2. False Report Warning
   User must confirm that the report is true to the best of their knowledge.

3. L1/L2 Withdraw Option
   Users can withdraw reports submitted in anger, confusion, or mistake.

4. Correction Option
   User can add correction without deleting the original statement.

5. Human Review Before Routing
   Serious action or escalation should require authorized review.

6. Evidence Credibility Check
   Evidence is used for context, not as final proof.

7. No Public Naming
   Reports are never publicly published, reducing revenge misuse.

8. High-Risk Safety Review
   L3/L4 withdrawal and closure require safety review.

Awaaz Setu does not show Aadhaar number in reports or normal dashboard view. Production implementation must follow legal consent, data minimization, encryption, audit logs, and authorized access controls.”
